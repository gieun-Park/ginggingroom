# 빠른 시작

## 1. 로컬 서버 실행

프로젝트 루트에서 다음 명령을 실행합니다.

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다. 카메라는 `localhost` 또는 HTTPS 환경에서만 사용할 수 있습니다.

## 2. 카메라로 촬영

1. 브라우저의 카메라 권한 요청을 허용합니다.
2. 촬영 버튼 위의 가로 프레임 목록에서 원하는 프레임을 고릅니다.
3. 선택한 프레임이 카메라 화면에 실시간으로 보이는지 확인합니다.
4. 필요하면 오른쪽 타이머 버튼을 눌러 **5초**를 켭니다.
5. 가운데 촬영 버튼을 누릅니다.
6. 결과를 **다운로드**하거나 **다시 찍기**로 돌아갑니다.

촬영 결과에는 카메라에 보이던 실제 배경과 선택한 프레임이 그대로 포함됩니다.

## 3. 이미지 업로드

카메라를 사용할 수 없거나 기존 사진을 쓰려면 왼쪽 **업로드**를 누릅니다. 선택한 사진은 외부로 전송되지 않으며 원래 배경을 유지합니다.

## 4. 테스트

```bash
npm test
npm run build
```

## 5. 자산 확인

```text
assets/
├── frames/
│   └── frame_01.png ... frame_26.png
└── models/
    └── face_landmarker.task
```

프레임 이미지는 투명 PNG를 사용하며, 얼굴 위치 조정 값은 `js/frame-config.js`에서 관리합니다.

## 문제 해결

| 문제 | 확인할 내용 |
|---|---|
| 카메라가 열리지 않음 | 사이트 카메라 권한과 HTTPS 또는 `localhost` 사용 여부 |
| 검은 화면이 보임 | 다른 앱이 카메라를 사용 중인지 확인한 뒤 새로고침 |
| 프레임이 안 보임 | 프레임 목록에서 선택 상태와 `assets/frames/` 파일 확인 |
| 얼굴 위치가 어긋남 | 정면을 바라보고 얼굴이 화면 안에 충분히 들어오게 조정 |
| 카메라 권한을 줄 수 없음 | **업로드**로 로컬 이미지 선택 |
| 다운로드가 안 됨 | 브라우저 다운로드 권한과 팝업 차단 설정 확인 |

모든 사진 처리와 얼굴 감지는 브라우저 내부에서만 실행됩니다.
